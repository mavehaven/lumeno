Image Storage — SQL Reference

Run these in Supabase SQL Editor to enable high-resolution image uploads in forms. Images are stored in a dedicated bucket with no size limit (configurable) and URLs are saved to responses.

1. Create Storage Bucket
-- Run in Supabase Dashboard → Storage → New Bucket
-- Name: "il-form-images"   Public: true   File size limit: 50MB
-- Or via SQL:
insert into storage.buckets (id, name, public, file_size_limit)
values ('il-form-images', 'il-form-images', true, 52428800)
on conflict do nothing;
2. Storage RLS Policy (allow authenticated uploads)
create policy "Allow authenticated uploads"
on storage.objects for insert to authenticated
with check (bucket_id = 'il-form-images');

create policy "Public read access"
on storage.objects for select to public
using (bucket_id = 'il-form-images');
3. Optional: Add image_urls column to responses
alter table il_responses
add column if not exists image_urls jsonb default '{}';
-- image_urls stores { "field_label": "https://..." } pairs
